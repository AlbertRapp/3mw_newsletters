Get Started with {crosstalk}

Everyone is talking about how you can use {shiny} apps without a server now using the powers of {webr}. That’s pretty cool. And I will talk about this in this newsletter at some point.

But today, I want to show you that R allowed you to build small interactive dashboards without a server for some time now. If you only need a few toggles to scroll through your data, then {crosstalk} can help you build just that. No Shiny server or long load times using {webr} necessrary.

Shared data with {crosstalk}

At the heart of using crosstalk lies a shared data set. This allows all the widgets that we throw into the dashboard to know the same things. Here’s how that works. First, we find prepare a data set that we want to share.

library(tidyverse)
data_prep <- economics |> 
  mutate(
    psavert = psavert / 100
  )

library(crosstalk)
shared_data <- SharedData$new(data_prep)

Preparing a slider

Once you have a shared data set, you can use it to bind it to some widget like a slider. You do that using the filter_slider() function from {crosstalk}. What this needs is

  • an ID
  • a label
  • a shared data set
  • a specification which column of the data is should target
# Don't forget the ~ to specify the targeted column
filter_slider("date", "Date", shared_data, ~date)

This will give you a slider similar to this one here.

Insert a plot

Now by the same logic we can add an image widget. Unfortunately, {crosstalk} doesn’t play well with {ggplot2}. But what it does play with is plotly. So let’s just create a ggplot and then convert it to a plotly chart with ggplotly().

We start with the ggplot. Nothing too fancy going on here. We can use the shared_data just like a normal tibble and then use some of it’s columns to build ggplot layers.

gg_plt <- ggplot(shared_data) + 
  geom_point(
    aes(date, psavert),
    alpha = 0.5,
    size = 2.5,
    col = '#78C2AD'
  ) +
  geom_smooth(
    aes(date, psavert), 
    linewidth = 2, 
    col = '#F3969A'
  ) +
  scale_y_continuous(labels = scales::label_percent()) +
  theme_minimal(base_size = 16) +
  labs(
    x = element_blank(), 
    y = 'Personal Savings Rate',
    title = '{crosstalk} lives!'
  )
gg_plt

And now we can convert it to plotly. Keep in mind that the automatic conversion is not always perfect but for a demo like this it will do. If you’re fluent in plotly, you could also just build the plotly chart from scratch.

library(plotly)
ggplotly(gg_plt)

Let’s make this interactive

Now, let’s bring our slider and the plot together. To do so, we can just wrap the slider and the plot into a bscols() function to align these things into a column format.

bscols(
  filter_slider("date", "Date", shared_data, ~date),
  ggplotly(gg_plt),
  widths = c(3, NA) # NA means fill remaining space
)

Where’s the line?

Nice, our interactive slider works and the points get filtered out quite smoothly. But our line sadly disappears when we don’t have the full data range. That’s a bit unfortunate which seems to be a problem with how geom_smooth() operates.

Thus, our workaround is to simply to that stuff manually. Hence, we fit a loess model ourselves.

loess_mod <- loess(
  psavert ~ seq_along(date), 
  data = economics
)

And then we use it as a prediction model in our data preparation.

data_prep <- economics |> 
  mutate(
    psavert = psavert / 100,
    loess_pred = predict(
      loess_mod, 
      seq_along(psavert)
    ) / 100
  )
shared_data <- SharedData$new(data_prep)

Finally, redoing our ggplot where we replace geom_smooth() by geom_line() will give us this code.

gg_plt <- ggplot(shared_data) + 
  geom_point(
    aes(date, psavert),
    alpha = 0.5,
    size = 2.5,
    col = '#78C2AD'
  ) +
  geom_line(
    # Setting this to a group is necessary or plotly won't plot
    aes(date, loess_pred, group = 1), 
    linewidth = 2, 
    col = '#F3969A'
  ) +
  scale_y_continuous(labels = scales::label_percent()) +
  theme_minimal(base_size = 16) +
  labs(
    x = element_blank(), 
    y = 'Personal Savings Rate',
    title = '{crosstalk} lives!'
  )

And consequently we get a smoother filtering experience

bscols(
  filter_slider("date", "Date", shared_data, ~date),
  ggplotly(gg_plt),
  widths = c(3, NA) # NA means fill remaining space
)

A table please.

A dashboard wouldn’t be a dashboard without some sort of table. And again, {crosstalk} cannot just use your favorite table package from R. But what it can work with is the DT package. So good for you if that’s your favorite.

We can throw in another column into our dashboard by wrapping DT::datatable(shared_data) into out bscols() function as well. Don’t forget to modify the widths in that case.

bscols(
  filter_slider("date", "Date", shared_data, ~date),
  ggplotly(gg_plt),
  DT::datatable(shared_data),
  widths = c(3, NA, NA) # NA means fill remaining space EQUALLY
)

Less columns please

After you have marveled at the magic of {crosstalk} you probably want to have only some of the columns in the table. The way to do that is to create a new shared data set for the table that has only the desired columns. And here’s the trick.

To make that work you want to tie together the original shared data and the new shared data in the same “group”. That way, the plot widget can still use the original data and the table can use the other data but both react to the slider simultaneously. Here’s how to set up the code.

shared_data <- SharedData$new(
  data_prep, 
  group = 'eco_data' # Set group name
)
shared_data_selected_cols <- SharedData$new(
  data_prep |> select(date, psavert), 
  group = 'eco_data' # Same name
)

Then, using this new shared data in the DT table gets the desired result.

bscols(
  filter_slider("date", "Date", shared_data, ~date),
  ggplotly(gg_plt),
  DT::datatable(shared_data_selected_cols), # New data here
  widths = c(3, NA, NA) 
)

Nice. The table is much conciser now. Let’s leave it like this for now. Next week, we’ll make the table a bit nicer and then wrap everything into a {bslib} dashboard style to make the table look nicer.
